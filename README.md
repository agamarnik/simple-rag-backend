# Simple RAG Backend

A lightweight Retrieval-Augmented Generation (RAG) API built with FastAPI, ChromaDB, and the Anthropic Claude API. Upload documents, then ask questions that get answered using only the content of those documents — with source attribution.

## How it works

This project combines three pieces:

1. **Chunking** — uploaded documents are split into overlapping text chunks (~500 characters each), so retrieval can work at a granular level rather than matching whole documents.
2. **Embedding + vector search** — each chunk is converted into a vector embedding and stored in a local ChromaDB collection. When a question comes in, it's embedded the same way, and ChromaDB's approximate nearest-neighbor search (HNSW) finds the most semantically similar chunks — not just keyword matches.
3. **Grounded generation** — the retrieved chunks are inserted into a prompt alongside the original question, and sent to Claude with an explicit instruction to answer *only* from that context. This keeps answers grounded in your actual documents instead of the model's general training knowledge.

Documents persist to disk (`./chroma_db`) via ChromaDB's `PersistentClient`, so the vector store survives server restarts without needing to re-embed everything.

## Setup

```bash
# Clone the repo
git clone git@github.com:your-username/simple-rag-backend.git
cd simple-rag-backend

# Create and activate a virtual environment
python3 -m venv venv
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Add your Anthropic API key
echo "ANTHROPIC_API_KEY=your-key-here" > .env
```

## Running the server

```bash
uvicorn main:app --reload
```

Visit `http://127.0.0.1:8000/docs` for interactive API documentation (auto-generated by FastAPI).

## API Endpoints

### `POST /upload`
Add a new document to the knowledge base.

**Request:**
```json
{
  "source": "eiffel_tower.txt",
  "content": "The Eiffel Tower was completed in 1889..."
}
```

**Response:**
```json
{
  "message": "Uploaded eiffel_tower.txt",
  "chunks_added": 1
}
```

### `POST /query`
Ask a question. Retrieves relevant chunks and generates a grounded answer.

**Request:**
```json
{
  "question": "When was the Eiffel Tower built?"
}
```

**Response:**
```json
{
  "answer": "The Eiffel Tower was completed in 1889.",
  "sources": ["eiffel_tower.txt"]
}
```

### `POST /chat`
Direct, non-RAG chat with Claude (no document retrieval).

### `GET /health`
Health check endpoint.

## Tech stack

- **FastAPI** — web framework and API layer
- **ChromaDB** — embedded vector database with built-in embedding + HNSW-based similarity search
- **Anthropic Claude API** — response generation
